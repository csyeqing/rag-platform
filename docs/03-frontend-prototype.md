# 前端原型说明（可运行）

## 1. 原型目标
提供可运行的 Web GUI 骨架，覆盖 MVP 全流程：
- 登录
- 模型配置
- 知识库管理
- 聊天问答
- 系统设置

## 2. 路由结构
- `/login`：登录页
- `/`：仪表盘
- `/providers`：模型配置
- `/knowledge-base`：知识库管理
- `/chat`：AI 聊天
- `/settings`：系统设置

## 3. 页面与功能

## 3.1 登录页
- 用户名/密码输入
- 登录成功后写入 token 并跳转

## 3.2 仪表盘
- 展示模型数、知识库数、会话数
- 显示系统关键策略摘要

## 3.3 模型配置页
- 创建 Provider 配置（OpenAI/Anthropic/Gemini/OpenAI-compatible）
- 在线配置验证
- 列表查看与删除

## 3.4 知识库页
- 创建私有/共享知识库
- 编辑知识库（名称、归属、标签）
- 删除知识库
- 目录同步
- 文件上传
- 文件列表与删除
- 重建索引
- 查看最近任务状态

## 3.5 聊天页
- 会话列表
- 选择模型配置和知识库
- 流式回答（SSE）
- 可切换引用展示和重排开关

## 3.6 系统设置页
- 默认语言、默认引用开关、默认 TopK（本地存储）
- 管理员用户管理（创建用户、启用/禁用、角色切换）

## 4. 状态管理
- Pinia `auth` store：token、角色、用户信息。
- 页面级 `ref/reactive`：Provider/KB/Chat 数据。

## 5. API 对接
- Axios 统一 `apiClient`。
- 请求拦截器自动注入 JWT。
- 聊天流式使用原生 `fetch + ReadableStream` 解析 SSE。

## 6. 视觉与交互风格
- 中文界面。
- 左侧导航 + 顶部状态栏。
- 登录页采用渐变背景与卡片化布局。
- Element Plus 负责表单、表格、按钮、消息提示。

## 7. 原型验收点
- 能从登录开始走通配置模型 -> 创建知识库 -> 上传文件 -> 发起对话。
- 聊天页面能看到流式文本增量。
- 引用开关可控制引用区块显示。
